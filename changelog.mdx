## Recent Updates

<Update label="2024-10-15" description="v2.0.0" tags={["breaking", "feature"]}>

## New Features

- Introduced Cases endpoint for managing form submissions: `GET /forms/{form}/cases`
- Added Categories endpoints for full CRUD operations: `GET /categories`, `POST /categories/create`
- New Modules endpoints to retrieve and manage items: `GET /modules/{module}/items`

## Breaking Changes

- Updated base path to `/api/site/v2`
- Deprecated v1.1.2 endpoints; migrate to v2 for continued support
- Authentication now requires both `apiKey` and `siteID` in headers

## Improvements

- Enhanced response formats with consistent pagination
- Added OAS 3.1 schema validation for better tooling support

</Update>

## Key Changes in v2.0.0

The v2.0.0 release represents a major update to the Siteglide API. You now interact with endpoints like Cases for form data, Categories for organization, and Modules for content management. All requests use the new server base: `{your-site}/api/site/v2`.

<Callout kind="alert">
  v1.1.2 is deprecated. Update your integrations immediately to avoid disruptions.
</Callout>

## Migrating from v1.1.2 to v2.0.0

Follow these steps to migrate smoothly.

<Steps>
  <Step title="Update Base URL" icon="globe">

    Change your API calls from v1 paths to the new v2 server:

````javascript
// Old
const base = 'https://api.siteglide.co.uk/v1';

// New
const base = `${yourSite}/api/site/v2`;
````

  </Step>
  <Step title="Add siteID Header" icon="settings">

    Include both `apiKey` and `siteID` in authorization headers:

````javascript
headers: {
  'apiKey': 'YOUR_API_KEY',
  'siteID': 'YOUR_SITE_ID'
}
````

  </Step>
  <Step title="Test New Endpoints" icon="play">

    Verify core endpoints like categories:

    <Request tabs="cURL,JavaScript">
    ````bash
    curl -H "apiKey: YOUR_API_KEY" \
         -H "siteID: YOUR_SITE_ID" \
         https://your-site.com/api/site/v2/categories
    ````
    ````javascript
    const response = await fetch('https://your-site.com/api/site/v2/categories', {
      headers: {
        'apiKey': 'YOUR_API_KEY',
        'siteID': 'YOUR_SITE_ID'
      }
    });
    ````
    </Request>

  </Step>
</Steps>
