## HTTP Status Codes

Siteglide API uses standard HTTP status codes to indicate the outcome of your requests. Review the table below to interpret responses quickly.

| Status Code | Meaning | Action |
|-------------|---------|--------|
| `200` | Success | Proceed with response data. |
| `201` | Created | Resource created successfully. |
| `400` | Bad Request | Check request parameters and format. |
| `401` | Unauthorized | Verify `api_key` and `site_id`. |
| `403` | Forbidden | Insufficient permissions for the action. |
| `404` | Not Found | Confirm endpoint and resource ID exist. |
| `429` | Rate Limited | Wait and retry after the specified delay. |
| `500` | Server Error | Contact support if persistent. |

<Callout kind="tip">
Always check the `message` field in error responses for specific details.
</Callout>

## Troubleshooting Authentication Failures

Authentication issues often stem from incorrect `api_key` or `site_id`. Follow these steps to resolve them.

<Steps>
  <Step title="Verify Credentials" icon="key">
    Ensure your `api_key` and `site_id` are correct. Find them in your Siteglide dashboard under **API Settings**.

    <CodeGroup tabs="cURL,JavaScript">
    ````bash
    curl "https://your-site.com/api/site/v2/categories?api_key=YOUR_API_KEY&site_id=YOUR_SITE_ID"
    ````

    ````javascript
    const response = await fetch('https://your-site.com/api/site/v2/categories?api_key=YOUR_API_KEY&site_id=YOUR_SITE_ID');
    ````
    </CodeGroup>
  </Step>

  <Step title="Test Basic Endpoint" icon="play">
    Call a simple endpoint like `GET /categories` to validate auth.
  </Step>

  <Step title="Regenerate Keys" icon="refresh-cw">
    If issues persist, regenerate your `api_key` in the dashboard and update your integration.
  </Step>
</Steps>

## Debugging Common Request Issues

Use these strategies for other frequent problems.

<Tabs>
  <Tab title="Rate Limiting" icon="clock">
    Implement exponential backoff in your code.

    <CodeGroup tabs="JavaScript,Python">
    ````javascript
    async function retryRequest(url, maxRetries = 3) {
      for (let i = 0; i < maxRetries; i++) {
        const response = await fetch(url);
        if (response.status === 429) {
          const retryAfter = response.headers.get('retry-after') || 1000 * Math.pow(2, i);
          await new Promise(resolve => setTimeout(resolve, retryAfter));
          continue;
        }
        return response;
      }
      throw new Error('Max retries exceeded');
    }
    ````

    ````python
    import requests
    import time

    def retry_request(url, max_retries=3):
        for i in range(max_retries):
            response = requests.get(url)
            if response.status_code == 429:
                retry_after = int(response.headers.get('retry-after', 1000 * (2 ** i) / 1000))
                time.sleep(retry_after)
                continue
            return response
        raise Exception('Max retries exceeded')
    ````
    </CodeGroup>
  </Tab>

  <Tab title="Invalid Parameters" icon="alert-triangle">
    Double-check path, query, and body parameters using `<ParamField>` docs.

    Common fix: Ensure IDs are integers, not strings.

    ```javascript
    // Wrong: string ID
    fetch('/modules/blog/items/abc123');

    // Correct: integer ID
    fetch('/modules/blog/items/123');
    ```
  </Tab>
</Tabs>
